3.2.8 、流程设置

【功能概述】

流程设置是根据企业业务流程管控的不同，设置系统提案管理流程的模块。

【操作说明】

1、通用区

程序路径：

系统管理---表单设置---流程设置

公共按钮：同通用表单操作事件；

2、专用区

启动：为选中功能节点的表单启用最新发布的工作流程，原有工作流程自动变为禁用状态；

停止：禁用选中的工作流程。在禁用前已经提交过的表单，还按原流程执行；新提交的表单按最新启动的工作流程执行。

流程模板设置：双击或者点击修改都可以进入到流程目标的设置界面。步骤如下：

1.      填写流程名称

2.      选择左面的流程节点类型：部门、岗位、职务、角色

3.      添加节点：选择流程节点，拖动到右侧的节点上，红叉为无效拖动。如下图所示

无效：

移除节点：将节点拖到右下角的垃圾箱为移除节点。

4.      保存流程，返回到流程设置界面，启动流程即可。

 

3、数据区

流程相关内容的展现，每个业务都可以具有多条流程管控，但是每个业务只能有一个流程启动，依据发布状态区别。

 启用状态  停用状态

        友好提示

1、流程中涉及到部门节点类型的，前提需要将部门岗位中的部门主管设置好。

流程中涉及到岗位节点类型的，前提需要将职员用户中的职务及主岗属性设置好。

2、流程启动后，业务已经开始走工作流，而且并未结束，此时如果修改了流程，已经走了工作流的单据是无法按照新的流程开展，因此建议不要轻易修改工作流。
